# One Piece Bounty Prediction (NLP + Network Analysis)
**Final Project (M.Sc. Data Science, HIT). Predicting the bounties of *One Piece* characters by combining textual signals with a character-relationship network, using CatBoost regression. Data was collected independently from the One Piece Fandom Wiki API — no Kaggle or pre-built dataset.**

## Overview
The "bounty" in *One Piece* is a quantitative threat score the story's government places on each dangerous character — a natural target to learn from text and from a network of relationships. This project predicts that bounty by engineering two complementary signals: textual features extracted from each character's wiki page, and structural features from a graph of connections between characters. The central finding is that text and network carry **complementary information — the combined model beats either one alone.**

## Key Features
- **Independent Data Collection:** Scraped 1,734 character pages from the [One Piece Fandom Wiki](https://onepiece.fandom.com/api.php) via its public API, yielding **218 characters with a known bounty** (the target) plus hundreds more used to build the relationship graph.
- **Text Features:** TF-IDF followed by SVD on the cleaned page text, with bounty figures scrubbed from the text before modeling to prevent leakage.
- **Network Analysis:** Built a character-relationship graph and extracted structural features; visualized with a layered k-core layout to keep a ~1,000-node graph readable.
- **Three-Way Model Comparison:** Trained and compared three models — **text-only, network-only, and combined** — against a baseline, evaluated with **R², RMSE, and MAE**.
- **Leakage Control:** TF-IDF and SVD fit on the training fold only within each CV split; identifier columns excluded from the model.
- **Error Analysis:** Investigated systematic errors (e.g. weak characters on strong crews being over-predicted), plus six visualizations.

## Tech Stack
- **Language:** Python
- **Modeling:** CatBoost, Scikit-learn
- **NLP:** NLTK, TF-IDF, Truncated SVD
- **Network:** NetworkX
- **Data Handling:** Pandas, NumPy
- **Visualization:** Matplotlib, Seaborn, adjustText

- `text_features.py`: Text cleaning and TF-IDF/SVD feature extraction.
- `network_plots.py`: Character graph construction and visualization.
- `features.py`: Feature assembly and helpers.
- `modeling.py`: Model training, cross-validation, and evaluation.
- `onepiece_raw.csv`: Raw collected dataset (~27MB, 1,734 rows).
- `one_piece_report.pdf`: Full project report.
- `one_piece_bounty_presentation.pdf`: Project presentation.
- `DATA.md` / `ETHICS.md` / `REFLECTION.md` / `AI_USAGE.md`: Data provenance, ethics, personal reflection, and AI-usage documentation.

## Dataset
The dataset was collected independently from the One Piece Fandom Wiki via its public API (content licensed CC BY-SA 3.0; used here for non-commercial academic purposes on fictional characters, so no personal data is involved). The raw file `onepiece_raw.csv` is included in this repository, and can be fully regenerated by running `data_collection.py`.
